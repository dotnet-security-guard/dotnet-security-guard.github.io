  Rules

The rules included in the extension.

Weak hashing function

The hashing function "X" is no longer recommended for password storage and signature generation.

Potential XPath injection with XmlDocument

The dynamic value passed to the XPath query should be validate

Event validation is disabled

Event validation is disabled. The integrity of client-side control will not be validate on postback.

Potential command injection with Process.Start

The dynamic value passed to the command execution should be validate.

Potential SQL injection with LINQ

The dynamic value passed in the SQL query should be validate.

Weak cipher algorithm

DES is not considered strong ciphers for modern applications. Currently, NIST recommends the usage of AES block ciphers instead of DES.

Weak random generator

The random numbers generated could be predicted.

View state mac is disabled (Future*)

View state mac is disable. The view state could be altered by an attacker. (This feature cannot be disabled in the recent version of ASP.net)

View state is not encrypted (Future*)

View state is not encrypted. Controls may leak sensitive data that could be read client-side.

Certificate Validation has been disable

Certificate Validation has been disable. The communication could be intercepted.

Weak cipher mode

The ciphertext produced is susceptible to alteration by an adversary. This mean that the cipher provides no way to detect that the data has been tampered with. If the ciphertext can be controlled by an attacker, it could be altered without detection.

CBC mode is insecure

This specific mode of CBC with PKCS5Padding is susceptible to padding oracle attacks. An adversary could potentially decrypt the message if the system exposed the difference between plaintext with invalid padding or valid padding.

Request validation has been disabled (Future*)

Request validation, which provided additionnal protection against Cross-Site Scripting (XSS), has been disabled.

ECB mode is insecure

An authentication cipher mode which provides better confidentiality of the encrypted data should be used instead of Electronic Codebook (ECB) mode, which does not provide good confidentiality. Specifically, ECB mode produces the same output for the same input each time. So, for example, if a user is sending a password, the encrypted value is the same each time. This allows an attacker to intercept and replay the data.





* The analysis of configuration files can be done, but Roslyn does not currently allow the reporting of error in static files.
